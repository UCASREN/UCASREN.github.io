LeetCode 15. 3Sum

LeetCode
  • Array
  • Medium
  • Two Pointers

Diffculty: Medium
URL: 3Sum

Question

Given an array S of n integers, are there elements a, b, c in S such that a + b + c = 0? Find all unique triplets in the array which gives the sum of zero.
Note: The solution set must not contain duplicate triplets.

Similar Problems:Two Sum、3Sum Closest、4Sum

Solution

我的思路：

  1. 几种解法的比较：
  2. 方法一：不打乱原数组顺序，为了将时间复杂度降低到$O(n^2)$，则需要使用到Hash。预先将数组里所有数存进Hash中，之后可以以$O(1)$的时间查找（由于可能存在重复数据，在Java中并没有找到合适的Hash数据结构，用HashMap的话只能这样存map.put(index, nums[index])，这样的话，查找只能用map.containsValue(nums[index])，然而通过源码可知map.containsValue(nums[index])的时间复杂度并不是$O(1)$，而是$O(n)$；而使用HashSet进行存储，查找的时间复杂度同样是$O(n)$）。
  3. 方法二：打乱原数组顺序，首先利用库函数对数组进行排序。由于数组是有序的，因此可以考虑使用二分查找，不使用Hash额外存储空间，这样下来时间复杂度为$O(n^2logn)$。
  4. 方法三：打乱原数组顺序，仍然先利用库函数对数组进行排序，然后利用Two Pointers思想进行查找，时间和空间复杂度分别是$O(n^2), O(1)$。
  5. 另外，还需要注意的是如何去重：
  • 第一个for循环遍历的时候，如果当前数和前面的数相等，那么就可以跳过此次循环进入下一次循环，因为当前数和后面的数能组成的结果肯定包含在前一个数组成的结果中了。
  • 另外，在下面的使用Two Pointers思想的while循环体内，如果找到了一组结果，并不马上跳出while循环，而是还需要继续找，但是此时也要避开重复的结果，也就是说首尾指针指向的数一定不能再等于当前指向的数（用一个while循环解决），最后还需要start ++; end --；
